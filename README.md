# 🚀 Clash.Meta (Mihomo) 智能订阅转换与动态分组 Worker

这是一个基于 Cloudflare Worker 的强大订阅转换脚本，一条链接，专为 Clash.Meta (Mihomo) 内核设计。只需一条链接就可以将各种常见协议的单节点链接或基础订阅，通过**远程 origin.yaml 模板**，全自动地转换为带有**地区智能分组**和**复杂路由策略**的高阶配置文件。

完全告别在模板中写死地区组的烦恼，实现真正的“有节点才生成，无节点不残留”！

---

## ✨ 核心特性

- 🔗 **多协议支持**：全面支持解析 `vmess`, `vless`, `ss`, `trojan`, `hysteria2 (hy2)`, `tuic` 协议。
- 🌍 **地区智能识别**：根据节点名称自动提取并归类地区（默认支持：🇯🇵、🇭🇰、🇸🇬、🇺🇸、🇹🇼、🇨🇦、🇬🇧）。
- ⚙️ **动态策略组生成**：根据实际存在的地区节点，自动生成对应的 `🚀最低延迟 (url-test)` 和 `♻️自动切换 (fallback)` 策略组。
- 🎯 **精准注入引擎**：无需在模板中预留死代码。脚本会自动将生成的地区策略组按设定规则（前面/后面）注入到指定的父级组（如：`✅节点选择`, `🕸GPT`, `🎧Spotify`）中。
- 📊 **流量信息透传**：自动提取原订阅的 `subscription-userinfo` 头，在客户端中正常显示流量使用情况。
- 📄 **纯模板驱动**：所有的路由规则 (rules) 和基础组配置均通过远程 `origin.yaml` 模板拉取，代码与配置分离，便于维护。

## 🛠️ 部署指南

1. 登录你的 [Cloudflare](https://dash.cloudflare.com/) 账号，进入 **Workers & Pages**。
2. 创建一个新的 Worker，并将本项目 `_worker.js` 中的完整代码复制并覆盖原内容。
3. （可选）在 Worker 的 **Settings -> Variables (环境变量)** 中，添加 `DEFAULT_TEMPLATE_URL` 变量，值为你托管在 GitHub 等平台的 `origin.yaml` 模板直链地址。
4. 点击 **Deploy (部署)**，你将获得一个 Worker 的域名（如 `https://sub.yourname.workers.dev`）。

## 📥 订阅地址参数说明

使用你的 Worker 域名生成最终订阅时，支持以下 URL 参数：

| 参数名称 | 是否必填 | 说明 | 示例 |
| :--- | :---: | :--- | :--- |
| `url` | **必填** | 原始节点或订阅的链接（需 URL 编码）。支持多个链接，用 `\|` 分隔。 | `?url=https://example.com/sub` 或 `?url=hvmess://...\|ss://...` |
| `name` | 选填 | 客户端中显示的配置文件名称，默认：`CF-SUB`。支持中文。 | `&name=我的极速节点` |
| `interval` | 选填 | 客户端自动更新订阅的间隔时长（单位：小时），默认：`24`。 | `&interval=12` |

**完整请求示例：**
`https://sub.yourname.workers.dev/?url=https://example.com/sub&name=自定义配置名&interval=12`

## ⚙️ 代码进阶配置说明

如果你想修改代码以适应你自己的需求，请关注代码顶部的 **核心配置区**：

### 1. 地区识别 (`REGIONS`)
通过修改关键词数组，你可以自定义脚本如何识别和归类节点。

### 2. 注入锚点 (`INSERT_AFTER_GROUP`)
动态生成的地区组（如 `🚀🇯🇵JP最低延迟`）会作为一个完整的区块，统一插在模板中某个指定策略组的后面。
*默认值：`🅰️Adobe` (请确保 origin.yaml 模板中存在此组)*

### 3. 智能注入规则 (`PARENT_GROUPS_CONFIG`)
这是本项目最强大的功能。你可以精准控制新生成的地区组如何分配到模板的父级组中：
- `target`: 你的 `origin.yaml` 中存在的父级组名称。
- `position`: `front` (插在target中其他静态节点前面) 或 `back` (插在后面)。
- `includeTypes`: 指定要注入的动态组后缀（如 `["最低延迟", "自动切换"]`）。
- `includeAllNodes`: 设置为 `true` 则会在该组最后自动追加所有的单节点。

## 📝 对 `origin.yaml` 模板的要求

为了让 Worker 能够完美工作，你的 `origin.yaml` 模板需要遵循以下原则：

1. **不需要写死地区组**：模板中**不要**包含如 `🚀🇯🇵JP最低延迟` 这样的硬编码地区组。
2. **需要提供父级容器**：预留好诸如 `✅节点选择`、`🕸GPT` 这样的主功能组框架。
3. **锚点定位**：模板中必须存在与代码中 `INSERT_AFTER_GROUP` 变量值一致的策略组名。
